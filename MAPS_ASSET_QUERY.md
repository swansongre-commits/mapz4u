# MAPS_ASSET_QUERY.md — MAPS 프로젝트 내 실행용 자산 조회 지시서

> **이 문서는 MAPS 프로젝트 폴더에서 Claude Code에게 넘기는 조회 전용 지시서다.**
> MAPZ가 아니라 **MAPS 프로젝트 안에서** 실행한다. 목적은 "무엇을 MAPZ로 넘길지"를 MAPS 쪽 실제 파일 기준으로 확정하는 것.
> 참조: `MAPS_to_MAPZ_자산이관_판단표.md`(원칙·이관 기준 원전) — 본 문서는 그 판단표를 이 프로젝트에서 실행하기 위한 질의문이다.

---

## 실행 방법

```bash
# MAPS 프로젝트 루트에서
claude
> "MAPS_ASSET_QUERY.md를 읽고 Step 1~4를 순서대로 실행해 MAPZ_ASSET_MANIFEST.md를 작성하라."
```

## Step 1. 프로젝트 구조 조회

1. `find . -maxdepth 3 -type f \( -name "*.faiss" -o -name "*.index" -o -name "*.pkl" -o -name "*.npy" -o -name "*.json" -o -name "*.csv" -o -name "*.py" \)` 로 후보 파일 전수 목록화
2. `.gitignore`에 걸려 안 보이는 대용량 파일(임베딩·인덱스류)이 있는지 별도 확인 (`du -sh` 로 용량 큰 파일 탐지)
3. README, CLAUDE.md, 노트북(.ipynb) 등 문서류에서 "직업", "학과", "임베딩", "FAISS", "커리어넷", "RRF", "동사" 키워드 검색(`grep -ril`)해 관련 파일 교차 확인

## Step 3. 이관 판단표 §3 기준 적용

Step 1에서 6개 슬롯 외에 발견된 파일은 아래 4문항으로 즉시 분류한다.

1. 데이터인가 로직인가? → 로직은 원칙적 제외(단, RRF 융합 등 헌장 중립 계산은 예외)
2. 그대로 노출 시 금지 목록(등급·매칭·연봉·전망) 위반인가? → 위반 시 "가공 후 이관"으로 표시
3. 개인정보(학생 식별정보) 포함하는가? → 포함 시 이관 제외
4. MAPZ MVP(주제 5개: 공룡·우주·동물·로봇·그림)에 지금 필요한가? → 불필요하면 "V2 보류"로 표시

## Step 4. 산출물 작성

`MAPZ_ASSET_MANIFEST.md`를 MAPS 프로젝트 루트에 작성한다. 구성:

1. Step 2 표 완성본 (실제 경로 채움)
2. Step 3에서 추가 분류된 파일 목록 (파일명 / 판정 / 사유)
3. **복사 명령 목록** — 각 이관 확정 파일에 대해 `cp <원본경로> <mapz경로>/data/maps_assets/<파일명>` 형태로 실행 가능한 명령어 나열 (이동 아님, 원본 보존)
4. 미해결 항목 — 슬롯에 해당 파일이 없거나 애매한 것들 (사람 확인 필요 시 여기에만 표시, 그 외는 무인 진행)

**주의**: 이 Step 4까지는 조회·매핑·명령어 생성만 하고, 실제 `cp` 실행과 MAPZ 프로젝트 반영은 MAPZ 쪽 AUTOPILOT.md Phase 0에서 수행한다. 이 문서는 MAPS 프로젝트를 건드리지 않는(read-only) 조회 전용 문서다.
