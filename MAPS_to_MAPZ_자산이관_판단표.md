# MAPS → MAPZ 자산 이관 판단표

- **작성일**: 2026-07-03
- **목적**: 기존 MAPS(고교 진로매칭 시스템) 프로젝트의 산출물 중 MAPZ(어린이 체험지도)로 가져올 것을 판단하는 체크리스트. Claude Code가 MAPS 프로젝트 폴더 구조를 조회한 뒤, 이 표의 기준으로 실제 파일명·경로를 확정해 `mapz/data/maps_assets/`로 복사한다.
- **원칙**: MAPS와 MAPZ는 목적함수·금지 규칙이 다르다(MAPZ는 판정·매칭 금지). **가져오는 것은 "재료"이지 "기능"이 아니다.** 매칭 알고리즘·추천 로직 자체는 이관 대상이 아니고, 그 밑에 깔린 벡터·데이터만 이관한다.

---

## 1. 이관 대상 (가져온다)

| # | 자산 | MAPS에서의 역할 | MAPZ에서의 역할 | 판단 근거 |
|---|---|---|---|---|
| 1 | 직업 임베딩 벡터 (549건) | 학생 흥미 ↔ 직업 매칭 | Layer A 그래프 엣지 (화면 비노출), '옆으로 한 칸' 인접도 계산 원료 | 벡터 자체는 중립적 데이터. 매칭이 아니라 인접성 계산에만 사용 → 헌장 위반 없음 |
| 2 | 학과 임베딩 벡터 (504건) | 학생 흥미 ↔ 학과 매칭 | Layer C(중등·부모 화면)용, 고1 과목 내비게이션 접속 시 원료 | 동일 논리. MVP에서는 미사용, V2 이후 활성화 |
| 3 | FAISS 인덱스 파일 | 최근접 탐색 | 동일 인덱스 재사용 또는 재구축 시 참조 | 재구축 비용 절감 목적. 스키마(§4.1 노드 5종)에 맞게 재색인 필요 |
| 4 | 커리어넷 원천 데이터(직업명·학과명·설명 텍스트) | 매칭 결과 표시 | Layer B 변환의 **원본 소스** — 연봉·전망 필드는 제거, 직업명은 동사 서술로 변환 후 사용 | 원본 텍스트가 있어야 변환 규칙(§4.3)을 적용할 수 있음. **가공 없이 그대로 노출 금지** |
| 5 | 임베딩 모델·전처리 파이프라인 코드 (BGE-M3 + RRF 융합 로직) | 하이브리드 검색 | AUTOPILOT Phase 3(연계방안 자동 결정)의 RRF 융합에 그대로 재활용 | 이미 검증된 코드, 재작성 불필요 |
| 6 | 동사 태깅 체계(있다면) | — | §4.2 동사 태그(7±2개) 설계의 참고 자료 | MAPS에 유사 체계가 있으면 재사용, 없으면 O*NET 기준 신규 설계 |

## 2. 이관 제외 대상 (가져오지 않는다)

| # | 자산 | 제외 이유 |
|---|---|---|
| 1 | 매칭·추천 알고리즘(학생→직업 직접 매칭 로직) | MAPZ 헌장 위반(판정·매칭 금지). 목적함수 자체가 다름 |
| 2 | 학생 개인 데이터(흥미검사 결과, 성적, 프로필 등) | 대상 연령·목적 불일치, 개인정보 이관 리스크. 애초에 필요 없음 |
| 3 | 등급·백분위·적성 유형 라벨 데이터 | 폐기·금지 목록 정면 위반 |
| 4 | 사용자 인증·계정 시스템 | MAPZ는 별도 서비스, 계정 체계 독립 설계 |
| 5 | UI/화면 코드 | MAPZ는 완전히 다른 화면 구조(4종 지도형 UI). 재사용 실익 없음 |
| 6 | MAPS 고유 비즈니스 로직(대입 전형 매칭 등) | 도메인 자체가 다름 |

## 3. 판단 기준 요약 (신규 자산 발견 시 적용)

MAPS 폴더 조회 중 표에 없는 파일을 발견하면 아래 질문으로 즉시 판단한다.

1. **이것이 "벡터/텍스트 데이터"인가, "알고리즘/로직"인가?**
   → 데이터면 이관 검토, 로직이면 대부분 제외 (단, RRF 융합처럼 헌장 중립적 계산 로직은 예외적으로 이관)
2. **화면에 그대로 노출되면 금지 목록(등급·매칭·연봉·전망 등)을 위반하는가?**
   → 위반하면 "가공 후 이관"(예: 항목 4), 위반 없으면 "그대로 이관"
3. **개인정보(학생 식별 가능 정보)를 포함하는가?**
   → 포함하면 이관 제외
4. **MVP(주제 5개: 공룡·우주·동물·로봇·그림) 범위에 필요한가?**
   → 불필요하면 V2 이후로 보류, 지금은 스킵

## 4. 실행 순서 (Claude Code 위임)

1. MAPS 프로젝트 폴더 구조 조회 (`find`/`ls`로 파일 목록 확보)
2. 위 §1 표의 6개 항목에 해당하는 실제 파일명·경로 특정
3. §3 판단 기준으로 표에 없는 추가 파일 분류
4. 이관 확정 파일을 `mapz/data/maps_assets/`로 복사 (원본은 MAPS 프로젝트에 그대로 유지 — 이동이 아니라 복사)
5. 이관 내역과 판단 근거를 `mapz/state/DECISION_LOG.md`에 기록 (AUTOPILOT.md의 결정 로그 규칙과 동일 형식)
