# 법적·개인정보 요건 (운영 §7)

MVP는 설계 문서만 있던 항목을, 운영버전에서 실제 구현/정책으로 전환한다.

## 1. 개인정보 — 비수집 원칙 (현재 구현)

- **무계정 익명 이용**: 로그인·회원가입·세션·쿠키 없음(코드상 인증 라우트 0, `app/main.py` 확인). DB에 사용자/이력/개인정보 테이블 없음(topic·experience·book·person·job·edge·recheck_queue만).
- 헌장 5조(기록은 아이의 것) + 개인정보보호법상 최소수집 원칙을 "아예 수집하지 않음"으로 구현.
- 계정 기능(예: "우리 아이 탐험 지도 저장")은 V2 이후 도입 시 아래 동의 플로우와 함께 구현.

## 2. 법정대리인 동의 플로우 (V2 도입 시)

- 만 14세 미만 개인정보 수집 시 개인정보보호법 제22조의2에 따라 법정대리인 동의 필요.
- 현재는 비수집으로 회피(익명 이용). 계정 기능 도입 시점에 동의 획득·검증 플로우 구현 예정.
- 개인정보보호위원회 「아동·청소년 개인정보 보호 가이드라인」 준수.

## 3. 공공데이터 출처 표시 (현재 구현)

화면 footer에 상시 고지(`app/templates/base.html`):
- 체험·행사: **한국관광공사 국문관광정보(TourAPI)** — 공공누리 제1유형(출처표시)
- 도서: **국립중앙도서관 정보나루 인기대출** — 저작자표시(BY), 출처 표시로 영리·변경·2차저작 자유이용
- 지도: **OpenStreetMap** — ODbL, © OpenStreetMap contributors

각 API 이용약관의 상업적 이용 가능 여부는 서비스 확장 시 재확인(공공누리 유형별).

## 4. [ ] 사람 확인 필요 (법무 검토 대기 — 무인 처리 불가)

- [ ] **위치기반서비스사업 신고**: 지도/위치 기능이 위치정보법상 신고 대상인지 법무 검토 필요.
- [ ] **약관·개인정보처리방침**: 실서비스 오픈 전 필수 문서, 법무 검토 필요.
- [ ] **상표·도메인**: MAPZ 상표 선등록(제9·41·42류)·도메인 가용성 확인(기획서 §1.1).

| 소스 | 라이선스 | 준수 사항 |
