## 4. Phase 0.5 하이라이트 — 실수요가 바꾼 것

- 채택 12주제 중 **MVP 5개(공룡·우주·동물·로봇·그림) 중 3개만 생존**(우주·동물·그림). **공룡·로봇 탈락**.
  - 이유: TourAPI 문화시설에 미술관 201·박물관 109개인데 공룡/로봇 전용 시설 희소 + 실대출도 픽션 시리즈에 밀림.
- 부상: 역사와위인(박물관 109·설민석 한국사 55,779대출), 과학실험, 몸과건강, 요리, 음악 등.
- 연령 프로파일도 실대출 기반(그림·우주=유아 피크, 역사·과학=초중 피크).

## 5. 운영 인프라 구현 (§3~7)

- **§3 데이터갱신**: `refresh.py`(일배치 오케스트레이터) + `freshness.py`(90일 재확인 큐 + 만료노출 감시, 현재 만료노출 0). `recheck_queue` 테이블.
- **§4 배포**: `deploy/Dockerfile`(healthcheck), `docker-compose.yml`(앱+nginx, DB 볼륨), `nginx.conf`(정적서빙·rate limit·TLS 슬롯).
- **§5 인증·보호**: 앱 rate limit(180/min/IP, 429) + nginx rate limit(10r/s) 2중. `docs/SECURITY.md`.
- **§6 제휴콘솔**: `admin/admin_app.py` — 관리자 Basic 인증(401/200 검증), 체험 CRUD·상태토글·최종확인일 자동갱신. 일반앱과 포트 분리.
- **§7 법적요건**: 출처표시 footer(TourAPI 공공누리1·정보나루BY·OSM), 비수집 원칙(무계정·PII테이블0). `docs/legal/`. 법정대리인·위치신고·약관은 [ ] 사람 확인.

## 8. 무인 진행 회고 (운영 트랙)
- **API 승인 반영**: 세션 중 사용자가 TourAPI·문화정보원 승인 → 재프로빙으로 KorService2 작동 확인, 즉시 실데이터 파이프 전환.
