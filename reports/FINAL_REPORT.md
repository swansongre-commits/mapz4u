## 9. 무인 실행 회고 — 사람이 개입했어야 할 뻔한 지점

이번 실험의 진짜 산출물. 무인 파이프라인이 §C 결정 규칙으로 자동 돌파한 지점들:

1. **API 키 전량 미활성** — data4library는 발급됐으나 `vitalizationErr`(활성화 대기), 문화정보원·TourAPI·박물관표준 서비스 미구독. 사람이라면 "키 승인 기다리자"며 멈췄을 지점. → MIXED로 자동 전환, 키리스 OSM + SEED로 완주.
2. **MAPS 임베딩 자산의 실체 부재** — 매니페스트가 상정한 BGE-M3/FAISS/RRF가 MAPS에 실제로 없었음(TF-IDF뿐). "재사용" 계획이 전제부터 어긋난 지점. → Layer A를 신규 TF-IDF 색인으로 재정의, DECISION_LOG에 명시.
3. **모델 다운로드 vs 재현성** — BGE-M3(Tier1)/MiniLM(Tier2)는 멀티-GB 다운로드로 무인 재현성 위험. → TF-IDF(Tier3) 선택. MAPS 계보와도 정합.
4. **디자인 '컨펌' 주체 부재** — design-reviewer 서브에이전트가 라운드1에서 금지어 '필독'을 잡아냄(부정문 안에 있어 사람도 놓치기 쉬움). → 3라운드 상한 내 자동 수정·재승인.
5. **자기참조 평가의 함정** — 만든 자가 채점하는 구조. 그런데 학부모 페르소나가 "필터 칩이 눌러도 안 걸러진다"는 실제 기능 결함을 독립적으로 잡아냄. → 즉시 수정·회귀테스트·재검증. 루브릭+증거 의무가 자기참조를 실질 검증으로 만든 사례.
6. **환경 특이 이슈** — Windows 콘솔(cp949)에 한글 stdout 파이프 시 프로세스 행(hang), collect_jobs 무한루프, 포트 8000 좀비 프로세스. 사람 개입 없이 진단·수정(리다이렉트/루프 가드/포트 소유 프로세스 종료)으로 돌파.

**결론: 무인 완주 성립.** 치명 환경오류 없이 Phase 0~9 전 게이트 통과, 로컬 기동 가능한 서비스 + 본 리포트 산출.
